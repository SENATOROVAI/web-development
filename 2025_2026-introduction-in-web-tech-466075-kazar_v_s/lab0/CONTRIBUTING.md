# Contributing

## Правила работы

- Рабочая ветка: `develop`.
- В `main` изменения попадают только через Pull Request.
- Каждый PR должен содержать описание изменений.
- Перед merge проверяются линтеры и базовая работоспособность.

## Коммиты

- Формат: короткое и понятное сообщение на английском.
- Пример: `Initial project setup`.
